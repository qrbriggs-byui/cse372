# Quiz: Turning User Insights into User Stories with Acceptance Criteria

This quiz tests your understanding of how to transform user feedback into actionable user stories and acceptance criteria.

---

**1.** What is the primary purpose of a **user story** in software development?  
A. To document all technical details of a feature  
B. To describe a feature from the user’s perspective, focusing on goals and value  
C. To define the final visual design of an application  
D. To list all testing requirements for a release  
**→ Correct answer:** B  

---

**2.** What is the standard format of a user story?  
A. When [condition], then [result]  
B. As a [type of user], I want [some goal], so that [some reason]  
C. Given [context], when [action], then [outcome]  
D. Who [user], does [action], for [result]  
**→ Correct answer:** B  

---

**3.** What is the main source of inspiration for creating user stories?  
A. Developer input and system requirements  
B. Marketing team feedback  
C. Insights from user surveys and low-fidelity prototypes  
D. Project management reports  
**→ Correct answer:** C  

---

**4.** What are the two main types of user information gathered from surveys and prototypes?  
A. Emotional and logical data  
B. Needs and frustrations; behavioral patterns  
C. Technical and business requirements  
D. Visual and textual content  
**→ Correct answer:** B  

---

**5.** What do **acceptance criteria** define?  
A. The marketing success of a product  
B. What needs to be true for a story to be considered complete  
C. The visual layout of a screen  
D. The scope of user research activities  
**→ Correct answer:** B  

---

**6.** Which of the following would be an example of a **good user story**?  
A. “The system should send a notification.”  
B. “As a student, I want to receive a confirmation message after reserving a study room, so that I know my booking went through.”  
C. “The interface should load in under two seconds.”  
D. “As a developer, I need to fix backend issues quickly.”  
**→ Correct answer:** B  

---

**7.** Why are user stories intentionally brief?  
A. To reduce project documentation  
B. To leave room for collaboration and discussion among the team  
C. Because they are mainly for testing purposes  
D. Because short stories are easier to present to clients  
**→ Correct answer:** B  

---

**8.** In the study room booking example, which of the following is an appropriate **acceptance criterion**?  
A. The confirmation message includes date, time, and room number.  
B. The user interface is visually appealing.  
C. The marketing campaign targets college students.  
D. The app icon must be blue.  
**→ Correct answer:** A  

---

**9.** What is the main benefit of pairing user stories with acceptance criteria?  
A. It adds creative freedom to the development process.  
B. It clarifies success conditions and makes testing easier.  
C. It eliminates the need for design documentation.  
D. It helps teams skip the prototyping phase.  
**→ Correct answer:** B  

---

**10.** Why is transforming feedback into user stories important?  
A. It turns user empathy and research findings into actionable development requirements.  
B. It speeds up coding by reducing team communication.  
C. It ensures the design is consistent with marketing goals.  
D. It allows stakeholders to skip user testing.  
**→ Correct answer:** A  

---
