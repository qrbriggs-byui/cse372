# Quiz 2-3: Turning Elicitation Data into Use Cases That Tell the User’s Story

## Question 1
**What is the main purpose of a use case in software development?**
- A. To define how the system is coded
- B. To describe how users interact with the system to achieve goals
- C. To document all technical constraints
- D. To provide a project budget breakdown  
✅ **Correct Answer:** B

---

## Question 2
**How do use cases bridge the gap between requirements gathering and development?**
- A. By translating human insights into structured, testable system interactions
- B. By replacing all stakeholder interviews
- C. By defining exact implementation details
- D. By focusing only on non-functional requirements  
✅ **Correct Answer:** A

---

## Question 3
**What does each use case primarily focus on?**
- A. How the system implements a function
- B. Who approved the requirement
- C. What the user wants to accomplish
- D. The financial cost of each feature  
✅ **Correct Answer:** C

---

## Question 4
**Which of the following is NOT typically included in a use case template?**
- A. Preconditions
- B. Postconditions
- C. Actor(s)
- D. Source code snippets  
✅ **Correct Answer:** D

---

## Question 5

- A. To request a refund

- C. To update their address
- D. To cancel a shipment

## Question 6
**What are “alternate flows” used for in a use case?**
- A. Describing unrelated features
- B. Capturing error conditions and variations from the main flow
- C. Listing user feedback
- D. Explaining the database schema

## Question 7
**Why is it important to validate use cases with stakeholders?**
- A. To make sure they understand programming logic
- B. To confirm the use case accurately reflects real user workflows
- C. To gain marketing approval
- D. To reduce documentation effort  
✅ **Correct Answer:** B

## Question 8
**When analyzing elicitation data, what should you look for to identify potential goals?**
- A. Repeated technical terms
- B. Statements like “I need to…” or “I usually…” from users
- C. Management’s strategic plan
- D. Error logs from the system

## Question 9
**Which of the following best describes the relationship between “main flow” and “alternate flow”?**

- B. Main flow describes normal interactions; alternate flow handles exceptions
- C. Both are interchangeable
