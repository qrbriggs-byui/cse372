# Software Requirements Quiz

## Question 1
**What is the main purpose of software requirements in the SDLC?**
- A. To describe how code should be written
- B. To ensure developers use the latest technologies
- C. To define what the system should do and how well it should perform
- D. To outline the project budget  
✅ **Correct Answer:** C

---

## Question 2
**According to the article, what happens when requirements are unclear?**
- A. Developers become more creative
- B. The team builds what was asked, but not what was needed
- C. Testing becomes easier
- D. The project finishes early  
✅ **Correct Answer:** B

---

## Question 3
**What is the key benefit of capturing regulatory and non-functional requirements early?**
- A. It improves marketing opportunities
- B. It avoids costly rework and project delays
- C. It increases the number of features delivered
- D. It allows skipping design reviews  
✅ **Correct Answer:** B

---

## Question 4
**How do requirements support testing and verification?**
- A. They provide a traceable link between features and tests
- B. They define the testing tools to be used
- C. They eliminate the need for user testing
- D. They specify only UI behavior  
✅ **Correct Answer:** A

---

## Question 5
**Why does the author compare writing requirements to “sharpening the saw”?**
- A. It helps teams avoid cutting corners
- B. It makes development slower but safer
- C. It’s an upfront investment that speeds up the rest of the process
- D. It emphasizes perfect documentation  
✅ **Correct Answer:** C
