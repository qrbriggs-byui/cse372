# Quiz: Communicating Requirements Effectively — Visual and Textual Strategies

This quiz covers key ideas from the reading on communicating requirements clearly using both visual and textual strategies.

---

**1.** Why is clear communication of requirements essential in software engineering?  
A. It ensures faster coding speeds  
B. It helps all stakeholders understand what the system should do  
C. It eliminates the need for testing  
D. It allows developers to skip documentation  
**→ Correct answer:** B  

---

**2.** What is the main advantage of combining visual and textual communication methods?  
A. It ensures every stakeholder understands the system regardless of background  
B. It makes presentations more colorful  
C. It eliminates the need for project management tools  
D. It replaces user testing entirely  
**→ Correct answer:** A  

---

**3.** Which of the following is an example of a textual communication tool?  
A. Wireframe  
B. Flowchart  
C. User story  
D. Journey map  
**→ Correct answer:** C  

---

**4.** What do **acceptance criteria** define?  
A. The visual layout of the interface  
B. The specific conditions that determine when a feature is complete  
C. The data model of the system  
D. The project timeline  
**→ Correct answer:** B  

---

**5.** Why are **visual communication tools** like wireframes and flowcharts valuable?  
A. They make logic easier to follow and help visualize system structure  
B. They replace the need for meetings  
C. They provide quantitative analysis of requirements  
D. They are used only by UX designers  
**→ Correct answer:** A  

---

**6.** Which of the following best describes a **use case**?  
A. A story told from a user's perspective about what they want to achieve  
B. A step-by-step description of how a user interacts with a system  
C. A diagram showing system data relationships  
D. A checklist of completed tasks  
**→ Correct answer:** B  

---

**7.** What does **tailoring your message** mean in the context of communicating requirements?  
A. Writing the same document for all audiences  
B. Adjusting how you present information based on your stakeholder’s background and priorities  
C. Limiting the number of visual aids you use  
D. Avoiding non-technical explanations  
**→ Correct answer:** B  

---

**8.** How can **emotional intelligence** improve requirement communication?  
A. By helping you adjust your communication style based on others’ reactions and preferences  
B. By making you appear more confident in meetings  
C. By increasing your technical vocabulary  
D. By minimizing the need for documentation  
**→ Correct answer:** A  

---

**9.** Which visual tool would best help explain how a process moves from one step to another?  
A. Flowchart  
B. Entity-Relationship Diagram (ERD)  
C. Journey Map  
D. Wireframe  
**→ Correct answer:** A  

---

**10.** What is the key takeaway from the reading about communicating requirements?  
A. Visuals are more important than text  
B. Text should always replace diagrams for precision  
C. The best communication combines visual clarity with textual precision to build shared understanding  
D. Developers should only communicate with other technical team members  
**→ Correct answer:** C  

---
